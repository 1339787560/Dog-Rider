## 概要

本文件是 `_Type-Definer.md` 的角色子类，覆盖 Creator-Client-DEV-xzmp（CocosCreator 3.8.1 TypeScript 客户端开发工程师）视角下的文档类型偏好。

**角色定位**：客户端业务逻辑开发者。负责 CocosCreator 3.8.1 下四川麻将客户端的 TS 脚本编写，包括游戏对局层、礼包插件层、UI 组件层。

**写作铁律**：
- **交互优先**：客户端的核心职责是"收消息 → 处理 → 发消息"。文档必须先讲清楚模块与外部（CPP 游戏服务、CP 礼包服务）的交互协议，再讲内部逻辑
- **源码驱动**：一切从实际源码出发，不虚构不杜撰。代码引用必须标注 `文件路径:行号`
- **事件追溯**：客户端采用事件驱动架构（eventCenter），遇到事件监听必须追溯到事件触发方和事件数据结构
- **由表及里**：先讲模块对外暴露的接口和交互方式，再深入内部状态管理和渲染逻辑

**外部服务交互关系：**

| 外部服务 | 交互方式 | 涉及模块 |
| :--- | :--- | :--- |
| **CPP 游戏服务**（GameConnect） | TCP 协议收发 | 游戏对局层：出牌、碰杠胡、结算等 |
| **CP 礼包服务** | HTTP/modsvr 调用 | 礼包插件层：迎新礼包、等级模块、活动等 |
| **客户端内部** | eventCenter 事件总线 | 模块间通信：手牌更新、牌桌状态、UI 刷新等 |

1. 程序接口文档：外部交互协议→事件定义→模块接口
2. 程序设计文档：插件架构→模块划分→状态管理→渲染流程
3. 程序详解文档：概要→外部交互→模块运行逻辑→数据结构→术语

## 附录

### 术语表

| 术语 | 含义 |
| :--- | :--- |
| **eventCenter** | 客户端事件总线，模块间通过 emit/listen 进行松耦合通信。发出方和监听方不需要直接引用对方 |
| **ViewCtrl** | MVC 模式中的视图控制器，管理 UI 组件的生命周期和业务逻辑绑定 |
| **Plugin** | 功能插件，客户端按功能划分的独立模块（40+ 个），每个插件有自己的 ViewCtrl/Help/View/Def |
| **GameConnect** | 网络连接管理器，统一处理与 CPP 游戏服务的 TCP 协议收发 |
| **GameInfo** | 游戏数据管理器（240KB+），存储对局状态、玩家信息、牌桌数据等全局状态 |
| **游戏对局层** | `assets/game/` 目录下的核心游戏逻辑，包括对局流程、手牌管理、牌桌交互等 |
| **礼包插件层** | `assets/plugins/` 目录下的 40+ 个业务插件，包括迎新礼包、等级模块、活动等 |
| **UI 组件层** | `assets/game/scripts/components/` 目录下的 UI 组件，如 3D 手牌、弃牌区、胡牌动效等 |
